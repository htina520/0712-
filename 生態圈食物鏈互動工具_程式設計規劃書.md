# 生態圈食物鏈互動工具：程式設計規劃書

## 1. 專案目標

製作一個給小學生使用的互動式網頁工具，讓學生透過按鈕選擇動物類型與動物個體，觀察該動物在生態圈食物鏈中的位置，以及牠和其他生物之間的關係，例如掠食、競爭、共生、分解等。

第一版先做「最小可運作版本」，使用單一 HTML 檔案，內含 CSS 與 JavaScript，不需要後端、不需要資料庫。

## 2. 使用者對象

- 國小學生
- 老師可在課堂中投影或讓學生用平板、手機操作
- 介面文字使用繁體中文
- 操作方式要簡單、按鈕大、文字清楚

## 3. 技術規範

- 單一檔案：`index.html`
- 內容包含：
  - HTML 結構
  - CSS 樣式
  - JavaScript 互動邏輯
- 不使用外部框架
- 支援 RWD：
  - 手機
  - 平板
  - 電腦
- 可直接用瀏覽器開啟

## 4. MVP 功能範圍

### 4.1 首頁主要畫面

畫面分為四個區域：

1. 標題區
   - 顯示「生態圈食物鏈探險」
   - 顯示一句簡短說明，例如「選一種生物，看看牠在食物鏈中的角色！」

2. 分類選擇區
   - 使用三個主要按鈕：
     - 按鈕 1：肉食動物
     - 按鈕 2：草食動物
     - 按鈕 3：分解者
   - 使用者點選後，下方顯示該分類的動物清單

3. 動物選擇區
   - 對應使用者的「按鈕 A 選擇動物」
   - 實作上可用大按鈕卡片呈現動物
   - 點選動物後，更新右側或下方資訊

4. 關係查看區
   - 對應使用者的「按鈕 B 決定想知道的關係」
   - 提供關係類型按鈕：
     - 全部
     - 掠食
     - 競爭
     - 共生
     - 被捕食
     - 分解
   - 點選後顯示目前選定生物與其他生物的關係

### 4.2 食物鏈層級顯示

顯示簡化的食物鏈層級：

```text
生產者 → 初級消費者 → 次級消費者 → 頂級消費者 → 分解者
```

每個層級用一個區塊呈現，並把目前選到的生物高亮顯示。

第一版範例層級：

- 生產者：草、樹葉
- 初級消費者：兔子、鹿、蚱蜢
- 次級消費者：青蛙、蛇
- 頂級消費者：老鷹、老虎
- 分解者：蚯蚓、菇類、細菌

## 5. 建議資料模型

在 JavaScript 中建立固定資料陣列，不從外部讀檔。

```javascript
const organisms = [
  {
    id: "rabbit",
    name: "兔子",
    category: "草食動物",
    level: "初級消費者",
    emoji: "🐰",
    description: "兔子主要吃草和嫩葉，是許多肉食動物的食物來源。"
  }
];
```

關係資料可另外建立：

```javascript
const relationships = [
  {
    from: "rabbit",
    to: "grass",
    type: "掠食",
    label: "兔子吃草",
    explanation: "兔子會吃草，所以兔子和草之間是取食關係。"
  },
  {
    from: "eagle",
    to: "rabbit",
    type: "掠食",
    label: "老鷹捕食兔子",
    explanation: "老鷹可能會捕食兔子，因此老鷹是兔子的掠食者。"
  }
];
```

## 7. 第一版建議關係資料

至少建立以下關係，讓每個分類都有可展示內容：

| 生物 A | 生物 B | 關係 | 顯示文字 |
| --- | --- | --- | --- |
| 兔子 | 草 | 掠食 | 兔子吃草 |
| 鹿 | 樹葉 | 掠食 | 鹿吃樹葉 |
| 蚱蜢 | 草 | 掠食 | 蚱蜢吃草 |
| 青蛙 | 蚱蜢 | 掠食 | 青蛙吃蚱蜢 |
| 蛇 | 青蛙 | 掠食 | 蛇吃青蛙 |
| 老鷹 | 蛇 | 掠食 | 老鷹捕食蛇 |
| 老虎 | 鹿 | 掠食 | 老虎捕食鹿 |
| 兔子 | 鹿 | 競爭 | 兔子和鹿都會吃植物，可能競爭食物 |
| 蚯蚓 | 落葉 | 分解 | 蚯蚓幫忙分解落葉，讓養分回到土壤 |
| 菇類 | 枯木 | 分解 | 菇類會分解枯木 |
| 細菌 | 動植物遺體 | 分解 | 細菌幫助分解死亡的動植物 |

可選的共生範例：

| 生物 A | 生物 B | 關係 | 顯示文字 |
| --- | --- | --- | --- |
| 樹 | 菇類 | 共生 | 有些菇類和樹根互相幫助取得養分 |

注意：若 MVP 不想增加「樹」這個生物，也可以暫時不做共生，或以「樹葉」代表植物來源，但文字要避免造成科學概念混淆。

## 8. 互動流程

### 預設狀態

- 預設選中「草食動物」
- 預設選中「兔子」
- 關係篩選預設為「全部」
- 畫面顯示兔子的描述、層級、關係清單與食物鏈位置

### 點選分類按鈕

1. 更新目前分類
2. 顯示該分類下的動物按鈕
3. 自動選取該分類的第一個動物
4. 更新資訊卡、關係清單、食物鏈層級

### 點選動物按鈕

1. 更新目前選定動物
2. 顯示動物名稱、分類、層級、簡介
3. 更新關係清單
4. 高亮食物鏈層級中的該生物

### 點選關係按鈕

1. 更新目前關係篩選
2. 顯示符合條件的關係
3. 若沒有資料，顯示友善提示，例如「目前還沒有這種關係資料」

## 9. 介面配置建議

### 電腦版

- 使用兩欄布局
- 左側：分類與動物選擇
- 右側：動物資訊與關係結果
- 下方：完整食物鏈層級

### 平板版

- 可維持兩欄，但欄寬縮小
- 按鈕保持足夠大

### 手機版

- 改為單欄垂直排列
- 按鈕可換行
- 食物鏈層級改成直向排列：

```text
生產者
↓
初級消費者
↓
次級消費者
↓
頂級消費者
↓
分解者
```

## 10. 視覺設計方向

- 適合小學生，明亮、清楚、不要太複雜
- 建議使用：
  - 淺綠色背景
  - 白色內容區
  - 黃色或橘色強調目前選取項目
  - 深綠色或深藍色文字，確保可讀性
- 按鈕要大，適合觸控
- 可使用 emoji 作為簡易生物圖示
- 不需要登入、不需要導覽列

## 11. 無障礙與可用性要求

- 所有按鈕使用 `<button>`
- 按鈕文字清楚，例如「1 肉食動物」
- 目前選中狀態需有明顯樣式
- 不只靠顏色判斷，也可加上「目前選擇」文字或勾選符號
- 字體大小至少 16px
- 手機按鈕高度建議至少 44px

## 14. 驗收標準

完成 MVP 後應符合：

- 可用瀏覽器直接開啟 `index.html`
- 手機、平板、電腦畫面不破版
- 可點選三種分類：
  - 肉食動物
  - 草食動物
  - 分解者
- 可點選動物並看到：
  - 名稱
  - 分類
  - 食物鏈層級
  - 簡介
- 可切換關係類型並看到關係說明
- 可看到完整食物鏈層級
- 目前選到的生物會在食物鏈中高亮
- 介面為繁體中文

## 15. 非 MVP 範圍

以下功能先不要做，避免第一版過大：

- 使用者自訂動物
- 儲存學習紀錄
- 後端資料庫
- 登入系統
- 複雜動畫
- 測驗模式
- 真實圖片素材
- 拖拉式食物鏈排序

## 16. 後續可擴充方向

- 加入「測驗模式」
- 加入更多生態系，例如森林、海洋、草原
- 加入圖片或插圖
- 加入「把生物拖到正確層級」活動
- 加入音效或朗讀
- 老師可切換簡易版與進階版
- 製作食物網，而不只是單一路徑食物鏈

## 17. 給實作 Agent 的提醒

- 第一版重點是「可以互動、可以理解食物鏈層級」，不要追求大量資料。
- 請保持程式碼清楚，資料與渲染邏輯分開。
- 若要加入 emoji，請確認在不同裝置上仍能閱讀；emoji 只是輔助，不應是唯一資訊。
- 若資料中有科學概念不確定，請使用保守、簡化且適合小學生的說法。
- 請優先完成可操作的 `index.html`，再調整視覺細節。
